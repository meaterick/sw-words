## 1. 기본 원칙 (General Principles)

- **가독성 우선**: 코드는 사람이 읽기 위해 작성한다. 과도한 축약이나 한 줄에 여러 로직을 욱여넣는 방식은 피한다.
- **보이스카우트 규칙**: 코드를 수정할 때 기존 코드가 조금이라도 더 나아지도록 개선한다.
- **YAGNI**: 지금 당장 필요 없는 기능이나 과도한 추상화는 구현하지 않는다.

---

## 2. 네이밍 규칙 (Naming Conventions)

| 대상 | 표기법 | 예시 |
| --- | --- | --- |
| 클래스 | PascalCase | `QuizHistory`, `WrongAnswerNote` |
| 함수 / 메서드 | snake_case | `generate_quiz()`, `check_answer()` |
| 변수 | snake_case | `user_answer`, `quiz_id` |
| 상수 | UPPER_SNAKE_CASE | `MAX_QUESTIONS` |
| 파일명 | snake_case | `main.py` |

- 함수는 동사로 시작하여 행동을 명확히 한다. (예: `get_questions()`, `save_record()`)
- 불리언 변수는 `is_`, `has_` 접두어를 사용한다. (예: `is_correct`, `has_token`)
- 누구나 알 수 없는 축약어는 사용하지 않는다.

## 3. 코드 스타일 & 포맷팅 (Code Style & Formatting)

- **들여쓰기**: 스페이스 4칸 (탭 사용 금지) — Python PEP 8 표준을 따른다.
- **줄 길이**: 한 줄은 최대 100자를 넘지 않도록 권장한다.
- **빈 줄**: 클래스 정의 사이에 빈 줄 2개, 메서드 사이에 빈 줄 1개를 둔다.
- **문자열**: 큰따옴표(`"`)로 통일한다.

```python
# ⭕ 올바른 예
class QuizHistory:
    def save_record(self, quiz, score):
        pass

    def view_record(self):
        pass


class WrongAnswerNote:
    def save_wrong_question(self, question):
        if question not in self.wrong_questions:
            self.wrong_questions.append(question)
```

## 4. 함수 및 메서드 설계 (Function Design)

- **단일 책임 원칙(SRP)**: 하나의 함수는 하나의 일만 수행한다.
- **함수 길이 최소화**: 가급적 20줄 이내로 유지한다.
- **매개변수 제한**: 인자는 최대 3개 이하로 유지한다.

```python
# ❌ 매개변수가 너무 많음
def create_user(name, age, email, address, phone):
    pass

# ⭕ 객체로 묶어서 전달 (Python에서는 딕셔너리 또는 dataclass 활용)
def create_user(user_info: dict):
    pass
```

## 5. 에러 핸들링 & 주석 (Error Handling & Comments)

### 5.1. 에러 처리

- 예외가 발생할 수 있는 곳은 `try-except` 블록으로 처리한다.
- `except` 블록을 비워두는 것은 금지한다. 최소한 `print` 또는 로깅을 남긴다.

```python
# ⭕ 올바른 예
try:
    score = int(input("점수를 입력하세요: "))
except ValueError:
    print("올바른 숫자를 입력해 주세요.")
```

### 5.2. 주석 (Comments)

- '무엇(what)'이나 '어떻게(how)'를 설명하는 주석은 피한다. 코드 자체가 말해줘야 한다.
- 주석은 '왜(why)' 이렇게 작성했는지 배경이나 특수한 조건을 설명할 때만 사용한다.
- AI가 생성한 코드 블록 상단에는 `# AI-generated` 주석을 추가한다.
- 남은 작업은 `# TODO: 할 일 내용` 형식으로 명시한다.

```python
# 대소문자 무시 및 앞뒤 공백 제거 — 사용자 입력 오류 허용을 위해 의도적으로 처리
def check_answer(self, user_answer: str) -> bool:
    return self.answer.strip().lower() == user_answer.strip().lower()
```

## 6. Git & Pull Request 규칙

- 커밋 메시지는 [Conventional Commits](https://www.conventionalcommits.org/) 표준을 따른다.
  - `feat:` 새로운 기능 추가
  - `fix:` 버그 수정
  - `docs:` 문서 수정
  - `refactor:` 코드 리팩토링 (기능 변화 없음)
- `main` 브랜치에 머지하려면 최소 1명 이상의 팀원 검토(Approve)가 필요하다.

## 7. 기타

- 입력값은 `strip()`으로 앞뒤 공백을 제거한 후 처리한다.
- 불필요한 `else` 절은 최소화한다 (early return 패턴 권장).
- 클래스 구분을 위해 `# --- ClassName ---` 형태의 구분선 주석을 사용한다.
