# 회고 및 개선 사항

> 단어즈 팀 | 소프트웨어개론 | 2026년 1학기

---

## 1. 팀 전체 회고

### 잘된 점

- 역할(PM·분석가·설계자·개발자·QA/보안)이 명확하게 분리되어 중복 작업 없이 진행되었다.
- GitHub를 통한 문서 관리로 변경 이력을 추적할 수 있었고, 팀원 간 최신 파일 공유가 원활했다.
- M1에서 작성한 클래스 다이어그램이 `src/main.py` 구현 시 그대로 활용되어 설계와 구현의 일관성이 유지되었다.
- AI 도구를 활용하되 비판적 검토를 거쳐 반영하는 원칙을 전 단계에서 지켰다.

### 아쉬운 점

- 일부 문서(M2 설계 패턴, SOLID 검토)가 늦게 작성되어 M3 단계에서 일괄 보완이 필요했다.
- 회의록 작성이 불규칙하여 의사결정 이력이 충분히 남지 않았다.
- `generate_quiz()` 내 문제 데이터 하드코딩 구조가 OCP를 위반하고 있으나, 프로젝트 일정상 구조 개선을 이번 학기 내에 적용하지 못했다.

### 배운 점

- 요구사항 정의서(FR/NFR)를 명확히 작성해두면 이후 설계·구현 단계에서 방향을 잃지 않는다는 것을 경험했다.
- 팀 내 Cross-check(인스펙션)를 통해 개인이 놓친 오류를 서로 발견할 수 있었고, 혼자 작업하는 것보다 품질이 높아졌다.
- AI 도구는 초안 생성에 효과적이지만, 다이어그램 문법 오류 등 기술적 오류를 직접 검증하는 과정이 반드시 필요하다.

### 다음에 다시 한다면

- 주차별 회의록을 꾸준히 작성하여 의사결정 근거를 남길 것이다.
- 문서 작성을 각 단계 완료 시점에 즉시 진행하여 마지막에 몰리는 상황을 방지할 것이다.
- 과목 추가 확장성을 고려하여 초기 설계 단계에서 Factory Method 패턴 또는 데이터 분리 구조를 채택할 것이다.
