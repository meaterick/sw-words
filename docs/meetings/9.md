## 논의 내용

**안건 1. `src/main.py` 구현 결과 공유 및 PM 검토**

논의 내용:
박성주가 노트북으로 구현한 코드를 실행하여 보여줬다. Python 퀴즈와 수학 퀴즈가 정상적으로 생성되고, 오답 저장 및 재풀이 기능이 동작했다. 학습 기록도 점수와 날짜가 기록되어 FR-01~05 기능 전부가 구현되어 있음을 확인했다.

박인찬이 사전에 Claude를 통해 PM 관점에서 코드를 검토하는 방법을 알아봤다. AI 조언대로 클래스 이름과 주요 메서드가 클래스 다이어그램과 일치하는지를 직접 비교해봤다. 7개 클래스 이름과 주요 메서드가 다이어그램과 일치하는 것을 확인했다. 사용자 입력 오류 처리가 단순하다는 점은 전달했지만 현재 범위 내 유지하기로 했다.

결정 사항:
- FR-01~FR-05 구현 완료 확인
- 클래스 구조 다이어그램과 일치 확인
- 단순 입력 오류 처리는 현 범위 내 유지 (향후 개선 사항으로 기록)

---

**안건 2. 팀 내 Cross-check 인스펙션 방식 확정**

논의 내용:
안광우가 인스펙션 방식을 설명했다. 역할 교환으로 각자 자기 역할이 아닌 영역을 검토하는 방식이다. 분석가(안병규)↔개발자(박성주), 설계자(임태훈)↔QA/보안(안광우)으로 교차 검토하기로 했다. PM(박인찬)은 PM 산출물(WBS)을 분석가가 검토하기로.

검토 대상은 `src/main.py`, `docs/M2/UML 다이어그램.md`, `docs/M1/requirements.md`로 정했다. 다음 주 목요일(6월 12일) 전까지 각자 검토를 마치고 결과를 안광우에게 전달하면 취합하기로 했다.

결정 사항:
- 분석가(안병규) → 개발자 산출물(src/main.py) 검토
- 개발자(박성주) → 분석가 산출물(requirements.md) 검토
- 설계자(임태훈) → QA/보안 산출물(SOLID 검토 문서) 검토
- QA/보안(안광우) → 설계자 산출물(UML 다이어그램) 검토
- PM(박인찬) → WBS는 안병규가 함께 검토
- 결과 취합: 안광우, 마감 6월 12일

---

**안건 3. 남은 M3 산출물 일정 재확인**

논의 내용:
최종보고서 제출까지 시간이 2주 남은 상황에서 코딩 표준 문서, AI 활동 내역, 회고 문서가 아직 남아있음을 확인했다. 인스펙션이 끝나는 6월 14일 이후 바로 이 문서들을 완성하고, 박인찬이 최종보고서에 취합하기로 했다.

결정 사항:
- 코딩 표준 문서: 박성주, 6월 16일 완료
- AI 활동 내역 요약: 안광우, 6월 16일 완료
- 개인 AI 로그 M3: 전원, 6월 18일 완료
