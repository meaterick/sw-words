## 논의 내용

**안건 1. WBS 주차 및 담당자 확정**

논의 내용:
박인찬이 AI(ChatGPT)로 WBS 표 구조를 참고하여 초안을 가져왔다. AI 초안에는 '프로토타입 제작', '단위 테스트', '통합 테스트' 같은 항목들이 있었는데 과목 범위와 맞지 않아 빼기로 했다. 실제 강의 계획서의 주차를 기준으로 기획(5~8주), 설계(9~11주), 구현·검토(12~13주), 마무리(14주)로 나눠 직접 조율했다.

설계 단계에서 유스케이스 다이어그램은 분석가(안병규)가, 클래스 다이어그램은 설계자(임태훈)가 맡기로 했다. 임태훈이 UML을 많이 안 다뤄봤다고 해서 PM이 자료를 공유해 주기로 했다.

결정 사항:
- WBS 최종 확정: 기획(5~8주) → 설계(9~11주) → 구현(12~13주) → 마무리(14주)
- 설계 단계 UML 담당: 유스케이스-안병규, 클래스-임태훈
- 핵심 로직 구현 담당: 박성주
- Cross-check 인스펙션 담당: 안광우

---

**안건 2. 비용 산정(FP) 방식 논의**

논의 내용:
안병규가 FP(기능점수) 산정 방식을 설명해 주었다. EI, EO, EQ, ILF, EIF 구분을 우리 기능에 맞게 적용하는 방법을 같이 확인했다. 오답 DB를 별도 ILF로 볼지 사용자 기록 DB에 합칠지 논의가 있었는데, 기능적으로 독립적이므로 별도 항목으로 두는 것이 맞다고 결론 냈다.

결정 사항:
- ILF 4개 확정: 문제 DB, 사용자 학습 기록 DB, 오답 DB, 사용자 DB
- EIF는 외부 연동 없으므로 0개
- 안병규가 FP 최종 산정 후 docs/M1/FP.md에 업로드

---

**안건 3. M1 GitHub 업로드 계획**

논의 내용:
M1 제출 마감 전에 문서 취합이 필요하다. 각자 작성한 파일을 GitHub에 올리는 과정에서 충돌이 날 수 있어서 PM이 최종 취합 후 한 번에 올리는 방식을 택하기로 했다. 임태훈이 git commit 방법을 잘 모른다고 해서 박인찬이 채팅방에 간단한 커밋 방법을 정리해 공유하기로 했다.

결정 사항:
- M1 최종 취합은 박인찬이 담당, 4월 28일까지 docs/M1/ 폴더 완성 목표
- GitHub 커밋 가이드를 카카오톡에 공유
