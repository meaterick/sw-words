## 논의 내용

**안건 1. 팀 역할 분담 최종 확정**

논의 내용:
수업 중에 대략 이야기된 역할 배분을 이번에 공식 확정하기로 했다. PM은 박인찬이 맡고, 분석가는 안병규, 설계자는 임태훈, 개발자는 박성주, QA/보안은 안광우로 정했다. 안병규가 처음에는 개발자 역할을 희망했는데, 요구사항 정의 경험이 있어서 분석가 쪽이 더 적합하다는 의견이 나왔고, 본인도 수긍하여 변경하였다.

결정 사항:
- PM: 박인찬, 분석가: 안병규, 설계자: 임태훈, 개발자: 박성주, QA/보안: 안광우로 최종 확정
- 역할은 학기 중 변경하지 않는 것을 원칙으로 함

---

**안건 2. GitHub 저장소 개설 방법 및 초대**

논의 내용:
GitHub 저장소를 누가 어떻게 개설할지 정했다. 박인찬이 organization 대신 개인 계정 기반으로 저장소를 만들고, 나머지 팀원을 collaborator로 초대하는 방식을 택했다. 임태훈이 조직(organization) 방식을 제안했지만, 관리 복잡도를 고려해 개인 저장소 방식으로 결론 냈다.

결정 사항:
- 저장소명: `sw-words`
- 박인찬이 오늘 중으로 저장소를 생성하고, 팀원 GitHub 계정을 카카오톡으로 수집하여 초대
- main 브랜치 직접 push 금지 원칙은 이번 프로젝트에서는 유연하게 운영하기로 함

---

**안건 3. 앞으로 소통 방식 정리**

논의 내용:
카카오톡 단체 채팅방이 이미 있어서 그걸 계속 쓰기로 했다. 문서 작업은 Google Docs에서 초안 작성 후 GitHub에 Markdown으로 올리는 방식을 따르기로 했다. 회의는 주 1회를 기본으로 하고, 바쁜 시기에는 필요에 따라 추가 진행.

결정 사항:
- 실시간 소통: 카카오톡 단체 채팅방
- 문서 관리: GitHub (Markdown)
- 초안 작업: Google Docs
