> 다음 오류들을 발견하였다. `Inspection.md` 검토 일시가 14주차(6/21)로 기재되어 있으나 WBS 및 최종보고서와 달리 13주차(6/14)가 맞다고 지적하였다. 회고 문서와 최종보고서 §13에서 "M1에서 작성한 클래스 다이어그램"이 잘못된 표현이라고 하였다. 최종보고서 팀원 목록에서 안광우 학번이 임태훈과 같은 32263877로 기재된 것을 지적하였다. `다음 단계 계획` 작성일이 6/21로 되어 있는데 M2 완료 시점 문서이므로 5/30이 적절하다고 하였다. `코딩 표준 문서.md`가 Python 프로젝트임에도 JavaScript·TypeScript 기준으로 작성되어 있다고 지적하였다. 박인찬 AI 로그는 전반적으로 내용 일관성이 있고 회의록 참조 번호(#3, #8, #9)도 이상 없다고 평가하였다.

**비판적 검토:**
> 발견된 오류들은 원본 파일과 직접 대조하여 모두 실제 오류임을 확인하였다. 학번 불일치는 여러 파일을 교차 비교하여 올바른 값을 직접 판단하였다. 코딩 표준 문서는 AI가 Python 기준으로 재작성한 초안이 실제 구현 코드(`check_answer()` 대소문자 처리, `WrongAnswerNote` 중복 방지 등)와 일치하는지 확인한 후 반영하였다.

**팀/개인 수정 내용:**
> 오류 6건 전부 직접 파일을 수정하였다. 코딩 표준 문서는 Python PEP 8 기준으로 AI가 재작성한 초안에 실제 팀 구현 코드 예시를 반영하여 확정하였다. 수정된 파일: `Inspection.md`, `회고 및 개선 사항.md`, `M3_최종보고서_단어즈.md`, `FP.md`, `다음 단계 계획`, `코딩 표준 문서.md`.

**최종 활용 결과:**

## 성찰 일지

**이번 최종 정리에서 AI를 어떻게 활용하였나요?**
> M2 `다음 단계 계획` 초안 작성, M3 최종보고서 통합, 전체 문서 오류 점검, 회의록 작성에 걸쳐 총 4회 활용하였다. PM 역할 특성상 직접 코드를 작성하거나 설계하는 일보다 문서 구조 잡기와 취합 작업에서 AI를 적극 활용하였다.

**AI 활용에서 가장 도움이 된 순간은?**
> 최종보고서를 통합할 때 M1·M2 산출물에서 내용을 빠짐없이 끌어와 13개 항목 틀에 맞게 구성해주는 작업이었다. 분량이 많아 혼자 처음부터 작성했다면 누락이 생겼을 가능성이 높았다.

**AI 활용에서 주의가 필요했던 순간은?**
> 최종보고서 초안에 학번 오류, 클래스 다이어그램 출처 혼동 등 사실 관계 오류가 여럿 포함되어 있었다. AI가 그럴듯하게 채워 넣은 값을 검토 없이 그대로 두었다면 오류가 그대로 제출될 뻔했다. 생성된 문서는 반드시 원본 파일과 교차 확인하는 과정이 필요하다는 것을 다시 확인하였다.

**AI 없이 직접 해야 했던 작업은?**
> 학번 오류의 경우 어느 값이 맞는지 AI가 판단할 수 없어 직접 여러 파일을 비교하여 결정하였다. 최종 제출 파일 GitHub 업로드, 팀원 소감 취합, 인스펙션 수정 반영 여부 최종 판단 등 팀 내부 상황과 의사결정이 필요한 작업은 전적으로 직접 수행하였다.
